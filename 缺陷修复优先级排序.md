# 视频下载器缺陷修复优先级排序

> 基于用户视角的全面代码分析，按严重程度和影响范围排序。

---

## P0 - 立即修复（可能导致崩溃/数据丢失）

| # | 问题 | 涉及文件 | 影响 |
|---|------|----------|------|
| 1 | ~~M3U8 并发下载无限制，可能导致内存溢出/App 崩溃~~ ✅ 已修复 (2026-06-19) | `M3U8DownloadHandler.swift` | 新增 `AsyncSemaphore` actor 限制片段并发下载数为 6，使用 `CheckedContinuation` 实现异步等待，`defer { Task { await semaphore.signal() } }` 确保异常时释放信号量。编译通过。 |
| 2 | ~~M3U8 合并片段全部加载到内存，大文件会 OOM~~ ✅ 已修复 (2026-06-19) | `M3U8DownloadHandler.swift` | 将 `Data(contentsOf:)` 替换为 `InputStream` + 固定 256KB 缓冲区流式复制，内存峰值恒定不随视频大小增长。编译通过。 |
| 3 | ~~`deleteDownloadTask` 逻辑错误，已完成文件永远不会被删除~~ ✅ 已修复 (2026-06-19) | `VideoDownloadEngine.swift` | 先保存 `completedURL` 和 `isCompleted` 状态，再执行 `cancel()`，避免状态被修改后条件判断失效。编译通过。 |
| 4 | ~~批量删除不清理已完成文件~~ ✅ 已修复 (2026-06-19) | `BatchDownloadManager.swift` | 改用 `deleteDownloadTask` 替代 `cancelDownload`，确保已完成文件被清理。编译通过。 |

## 建议修复路线

### 第一阶段：稳定性（P0）
- M3U8 并发限制（信号量控制 5-10 个）+ 流式合并（`FileHandle`）
- 修复 `deleteDownloadTask`：先保存 `completedURL` 再 cancel
- 统一队列管理器：`BatchDownloadManager` 复用 `VideoDownloadEngine` 的 `queueManager`
- 集成 `DownloadTaskDatabase`：任务创建/状态变更时同步写入，启动时恢复

### 第二阶段：可靠性（P1）
- 集成 `BackgroundDownloadSession`，修复 `UIBackgroundModes` 配置
- 持久化 M3U8 下载状态（`completedSegments` 写入磁盘）
- 修复速度计算：使用实际下载字节数
- 文件下载增加重试逻辑（指数退避）
- 请求通知权限

### 第三阶段：体验（P2）
- 增强 M3U8 解析（密钥轮换、fMP4、VOD/Live 区分）
- 格式检测增加 HEAD 请求 Content-Type
- 添加 URL 输入框、设置页面、网络状态监听
- 修复 Tab 重复、添加暂停/恢复/重试 UI
- 批量创建容错（部分失败继续处理其余项）
- ATS 配置白名单域名

### 第四阶段：完善（P3）
- 文件管理页、视频预览、暗色模式、国际化
- 清理死代码和调试语句
- 优先级队列、更多格式支持
- 数据库 WAL 模式和事务保护
